# Notes

This directory is the working memory for the repo.

## Structure

- `notes/plans/` = active work; current scope, status, and next steps live here
- `notes/journal/` = dated checkpoints; use for milestones, reversals, and retrospectives
- `notes/research/` = reusable analysis; use for comparisons, literature, and design reasoning
- `notes/archive/` = stale or superseded material; move old scratch notes here

## Workflow

- Start with `notes/plans/` when deciding what to do next.
- Keep the current truth in the relevant plan note, not in a top-level index.
- Add a `notes/journal/` entry only when something meaningful happened.
- Add to `notes/research/` when the writeup is reusable beyond one task.
- Move stale notes to `notes/archive/` instead of endlessly curating this file.
- If someone asks what happened so far, synthesize from the active plan plus recent journal entries.

## Notes on maintenance

- This file should describe the structure and workflow only.
- Do not turn this into a hand-maintained list of note filenames.
- Keep raw logs in `logs/`; keep durable summaries and lessons here.
